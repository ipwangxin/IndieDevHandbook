# 工具清单

> *字数：约 3610 字；预计阅读时间：8 分钟。*


这份清单会持续更新，优先收录独立开发早期真正用得上的工具。

选择工具时，不要只看技术上是否先进。早期更重要的是：便宜、稳定、上手快、方便迁移、社区资料多。

## 写作和文档

### VitePress

适合写免费书、文档站、教程站和产品说明。

优点：

- Markdown 写作，维护成本低。
- 默认主题够用。
- 静态部署简单。
- 适合长期更新。

### GitHub

适合托管代码、内容、Issue、Discussions 和开源项目。

如果你的受众是开发者，GitHub 本身也是一个展示入口。

### Giscus

适合给文档站接入评论。

它基于 GitHub Discussions，不需要自己维护数据库，适合开源书、文档和案例库。

## 部署

### Vercel

适合 Next.js、VitePress、静态站和前端项目。

上手快，适合早期验证。国内访问速度可能不稳定，需要根据用户群体判断。

### Cloudflare Pages

适合静态站和前端项目。

全球 CDN 强，免费额度友好，也适合部署文档站和落地页。

### GitHub Pages

适合最简单的静态站。

优点是免费、稳定、和 GitHub 仓库天然集成。缺点是动态能力较弱。

## 数据库和后端

### Supabase

适合快速做账号、数据库、存储和简单后端能力。

对独立开发者友好，但国内访问和合规要根据项目情况评估。

### Neon

适合托管 PostgreSQL。

如果你的项目已经确定使用 PostgreSQL，可以考虑。

### 自建服务器

适合需要国内访问速度、备案、支付回调、文件存储或更可控环境的项目。

缺点是运维成本更高。

## 数据和统计

### Umami

轻量、隐私友好的访问统计工具。

适合独立开发者看访问、来源、页面表现。

### Plausible

轻量统计工具，体验好，但通常需要付费或自托管。

### Google Analytics

功能完整，但对早期项目可能偏重，国内访问也可能不稳定。

## AI 开发和写作

### ChatGPT / Claude / Gemini

适合做产品定义、文案、代码解释、调试思路、SEO 标题和用户反馈整理。

早期最有用的不是让它直接生成完整系统，而是让它帮你把模糊想法变清楚：

- 目标用户是谁？
- 第一版只保留哪条核心路径？
- 落地页第一屏怎么写？
- 用户反馈里有哪些重复问题？
- 这段代码有什么安全和成本风险？

### Cursor / GitHub Copilot / Codex 类工具

适合在已有项目里改代码、补组件、修 bug、写测试和解释代码。

使用建议：

- 先让它读现有文件，再让它改。
- 每次只改一个明确目标。
- 让它说明改了哪些文件。
- 关键逻辑自己再读一遍。
- 支付、权限、隐私、删除、成本相关代码要特别检查。

### AI 图片和素材工具

适合做早期占位图、社交分享图、文章配图和简单图标灵感。

但正式产品的 logo、品牌主视觉和涉及版权的素材，还是要注意授权和可商用范围。

## 表单和反馈

### Tally

适合快速做海外用户表单、申请表、反馈表。

### 飞书表单

适合国内用户，填写门槛低，适合收集需求和报名。

### 腾讯问卷

适合更大众的国内用户。

### GitHub Issues

适合开发者用户和开源项目。

但如果你的用户不是开发者，Issue 门槛可能太高。

## 早期推荐组合

如果你不知道怎么选，先按阶段选，不要按“哪个工具最火”选。

| 阶段 | 目标 | 推荐组合 |
| --- | --- | --- |
| 0 元验证 | 先确认有没有人感兴趣 | VitePress/GitHub Pages + 飞书表单 + GitHub Discussions |
| 50 元以内 | 做一个能访问的落地页 | 域名 + Cloudflare Pages/Vercel + Google Search Console |
| 100-300 元 | 跑一个有真实用户的小产品 | 轻量服务器或 VPS + Umami + 数据备份 + 基础监控 |
| 有收入后 | 稳定运营和减少事故 | 服务器/数据库分离 + 对象存储 + CDN + 日志和告警 |

### 0 元验证组合

适合还不确定需求的时候。

```text
知乎 / V2EX / 即刻发帖
+ 飞书表单收集反馈
+ GitHub Pages 或 Cloudflare Pages 放一个说明页
```

这个阶段不要急着买服务器。你要验证的是：有没有人愿意点进来、留言、填表、给样本。

### 50 元以内上线组合

适合做免费书、工具说明页、等待名单和简单 MVP。

```text
域名
+ Cloudflare Pages / Vercel
+ Google Search Console
+ Giscus 或表单反馈
```

如果你的项目没有后端，这套就够用了。

### 100-300 元生产组合

适合已经有少量真实用户、需要稳定服务的小产品。

```text
轻量服务器 / VPS
+ 数据库备份
+ Umami 或其他访问统计
+ Uptime 监控
+ 基础日志
```

这个阶段最怕的不是架构不够高级，而是服务挂了没人知道、数据坏了没有备份。

### 有收入后的升级组合

等产品有收入或稳定用户后，再考虑拆分：

- 应用服务器和数据库分开。
- 文件放对象存储。
- 静态资源走 CDN。
- 加错误监控和告警。
- 给支付、登录、核心任务单独做日志。

有收入后再追求稳定性，心态会健康很多。

如果你要做一本免费书：

```text
VitePress + GitHub + Vercel/Cloudflare Pages + Giscus
```

如果你要做一个轻量工具：

```text
Next.js/Vue + Supabase/SQLite + Vercel/云服务器 + Umami
```

如果你要先验证需求：

```text
知乎文章 + 飞书表单 + 微信群 + 简单落地页
```

## 部署工具快速推荐

如果你只是测试，可以优先用免费平台：

| 场景 | 推荐 | 备注 |
| --- | --- | --- |
| 免费书 / 文档站 | Cloudflare Pages、Vercel、Netlify | 适合 VitePress、静态站、落地页 |
| Next.js Demo | Vercel | 上手最快，适合 MVP |
| 普通后端 Demo | Render | 免费层适合测试，但注意休眠 |
| 开源项目展示 | GitHub Pages | 简单稳定，适合纯静态 |
| 国内用户生产 | 阿里云、腾讯云、华为云轻量服务器 | 注意备案、续费价、备份 |
| 免备案中文站 | 香港服务器 | 注意线路、带宽和稳定性 |


## SEO 和增长工具

早期不需要买很多昂贵工具，先把基础设施补齐。

| 需求 | 推荐工具 | 备注 |
| --- | --- | --- |
| 访问统计 | Umami、Plausible、Google Analytics、Cloudflare Web Analytics | 先知道用户从哪里来 |
| 搜索收录 | Google Search Console、Bing Webmaster Tools、百度搜索资源平台 | 上线后主动提交站点 |
| 关键词观察 | Google Trends、百度指数、知乎搜索、5118、Ahrefs Free Tools | 先用免费或低成本方案 |
| 页面检查 | Lighthouse、PageSpeed Insights | 检查速度、SEO、可访问性 |
| 评论反馈 | Giscus、GitHub Discussions、飞书表单、Tally | 把读者变成反馈来源 |
| AI 开发 | ChatGPT、Claude、Gemini、Cursor、GitHub Copilot、Codex | 加速第一版，但关键逻辑要自己检查 |

## 服务器预算速查

| 月预算 | 适合阶段 | 推荐方案 |
| --- | --- | --- |
| 0 元 | Demo、免费书、落地页 | Cloudflare Pages、Vercel、GitHub Pages |
| 30-100 元 | 个人项目起步 | 轻量服务器、香港 VPS、静态前端 + 小 API |
| 100-300 元 | 有真实用户的小产品 | 2 核 4G 起步，数据库备份，对象存储 |
| 300 元以上 | 稳定运营 | CDN、应用服务器、数据库、对象存储、监控拆分 |

选择原则：

- 测试阶段优先免费和省心。
- 生产阶段优先稳定、备份和续费价格。
- 国内用户多就认真考虑备案和国内服务器。
- 不想备案可以先用香港服务器过渡。
- 不要一开始为未来百万用户设计架构，先为前 100 个真实用户设计。

## 变现方式速查

| 变现方式 | 适合什么 | 早期建议 |
| --- | --- | --- |
| 广告 | 免费工具站、内容站、导航站 | 有稳定流量后再做 |
| 订阅 | SaaS、AI 工具、监控、持续服务 | 免费版 + Pro 起步 |
| 买断 | 模板、电子书、插件、小工具 | 写清楚更新和售后边界 |
| 按量付费 | AI、OCR、API、文件处理 | 用点数制覆盖成本 |
| 联盟推广 | 工具清单、教程、评测 | 只推荐和读者相关的服务 |
| 赞助 | 免费书、Newsletter、社区、开源项目 | 先证明读者垂直和稳定 |
| 服务咨询 | B2B、专业能力、复杂问题 | 适合早期验证高价值需求 |
| 数字产品 | 课程、模板、资料包、案例库 | 用免费内容引流 |
| 企业版 | 团队工具、B2B SaaS | 等出现发票、权限、私有化需求 |
| API/数据 | 数据库、监控、聚合能力 | 注意数据来源和合规 |
