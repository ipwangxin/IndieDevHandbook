# 检查清单

> *字数：约 760 字；预计阅读时间：2 分钟。*


这里放一些可以直接使用的检查清单。每次开始一个新项目、上线一个页面或准备推广时，都可以先过一遍。

## 点子验证

- 这个需求来自真实吐槽吗？
- 用户是谁，能不能具体到职业、身份或场景？
- 用户现在怎么解决？
- 这个问题是否重复发生？
- 用户是否已经付出时间、金钱或人工成本？
- 现有方案为什么不好？
- 这个问题能不能用一个网页 MVP 验证？
- 用户是否愿意留下联系方式？
- 用户是否愿意提供真实样本？
- 是否存在明确的付费或业务价值？

## MVP

- 第一版是否只验证一个核心路径？
- 用户是否能在 3 分钟内理解产品？
- 用户是否能在 5 分钟内完成第一次使用？
- 是否去掉了不必要的账号、权限和后台？
- 是否有明确的反馈入口？
- 是否能收集访问和转化数据？
- 是否保留了人工兜底空间？
- 如果验证失败，是否能快速调整方向？

## 上线前

- 域名已购买或已确定使用平台默认域名。
- 根域名、www 或二级域名的使用方式已经想清楚。
- DNS 解析已配置，A / CNAME 指向正确。
- HTTPS 能正常访问。
- HTTP 会跳转到 HTTPS。
- 证书不是浏览器不信任的证书。
- 自建服务器的证书续期已经配置。
- 国内服务器绑定域名时，备案状态已经确认。
- 页面标题和描述清楚
- 移动端可读
- 首页能在第一屏说明价值
- 有明确行动按钮
- 有反馈入口
- 有访问统计
- 404 页面不影响主流程
- 表单或提交入口已测试
- 关键链接可点击

## 推广前

- 这篇内容面向谁？
- 它解决用户的哪个具体问题？
- 标题是否足够具体？
- 开头是否先讲场景，而不是先讲产品？
- 是否有真实例子、截图或数据？
- 结尾是否只有一个行动按钮或动作？
- 这个动作是否足够轻？
- 是否准备好承接评论、私信或表单提交？

## 复盘

- 这次推广有多少曝光？
- 有多少点赞、收藏、关注？
- 有多少评论和私信？
- 有多少人访问网站？
- 有多少人完成核心动作？
- 哪句话最容易引发互动？
- 哪个渠道带来的反馈最具体？
- 下一次应该优化标题、内容、CTA 还是产品本身？
