# 开发资源和技术选择

> *字数：约 1353 字；预计阅读时间：3 分钟。*


早期开发的目标不是炫技，而是尽快把一个可验证的版本交到用户手里。

技术选择要服务产品阶段。还没验证需求时，优先选择熟悉、便宜、部署简单、社区资料多的方案。

## 要不要纠结技术栈

不要过度纠结。

技术栈会影响开发效率和维护成本，但大多数独立开发项目不是死于框架，而是死于没有需求、没有用户、没有转化。

你可以按这条原则选：

```text
熟悉 > 稳定 > 生态 > 性能 > 新潮
```

如果你已经熟悉 Vue，就用 Vue。熟悉 React，就用 React。熟悉 Laravel、Django、FastAPI、Go，也都可以。

真正要避免的是：为了一个还没验证的点子，花很多时间研究“最完美架构”。

## 常见项目怎么选框架

| 项目类型 | 推荐 | 说明 |
| --- | --- | --- |
| 免费书 / 文档站 | VitePress | 写 Markdown，部署简单 |
| SEO 内容站 | Astro、Next.js | 适合静态生成和长尾内容 |
| AI 工具站 | Next.js、Nuxt、FastAPI | 前端、API、文件上传比较好组织 |
| SaaS 后台 | Next.js、Laravel、Django、Rails | 鉴权、数据库、支付、后台更完整 |
| 管理后台 | Vue + Element Plus、React + Ant Design | 组件成熟，开发快 |
| 小脚本 API | FastAPI、Express、Hono | 适合快速做服务 |

## 免费开发资源

这些资源适合测试、原型和早期验证。

### 免费部署

- Cloudflare Pages：适合静态站、文档站、落地页。
- Vercel Hobby：适合个人项目、Next.js Demo、前端应用。
- Netlify Free：适合静态站和 Jamstack 项目。
- GitHub Pages：适合开源项目展示和纯静态站。
- Render Free：适合测试后端服务，但免费服务通常会休眠，不建议生产。

### 免费数据库和后端

- Supabase Free：适合快速做数据库、鉴权、存储原型。
- Neon Free：适合 PostgreSQL 测试。
- Upstash Free：适合 Redis、队列和轻量缓存测试。
- Firebase Free：适合简单应用和实时数据原型。

免费数据库适合验证，不适合长期保存关键生产数据。只要开始有真实用户，就要考虑备份和迁移。

### 免费域名

免费域名可以用来测试，但不建议作为正式生产域名。

可选方式：

- GitHub Student Developer Pack：学生可以拿到一些域名和云服务权益。
- EU.org：可以申请免费二级域名，但审核和稳定性要自己评估。
- 平台默认域名：Vercel、Cloudflare Pages、Netlify 都会给项目默认域名。

正式产品建议买一个自己的域名。域名是长期资产，不要为了省几十块钱影响信任感。

### 免费本地穿透

本地穿透适合开发支付回调、Webhook、演示本地服务。

常见选择：

- Cloudflare Tunnel
- ngrok
- localtunnel
- Serveo 类工具

建议：

- 开发测试可以用免费穿透。
- 支付回调、生产 Webhook 不建议长期依赖免费穿透。
- 生产环境要用正式域名和稳定服务器。

## 开发期间最低配置

一个轻量 MVP 至少要有：

- 一个可访问地址。
- 一个核心功能页面。
- 一个反馈入口。
- 一个访问统计。
- 一个错误查看方式。
- 一个数据备份方式。

如果涉及文件上传、AI 生成、支付、用户数据，就要额外考虑：

- 文件存储
- 任务超时
- 成本限制
- 失败重试
- 用户隐私
- 数据删除

## 有实力还是建议买服务器

免费资源适合测试，但正式上线后，有预算还是建议买服务器或付费托管。

原因很简单：

- 免费服务可能休眠。
- 免费额度可能变化。
- 免费域名不利于品牌。
- 免费数据库不适合长期生产。
- 出问题时排查和支持有限。

如果你的产品开始有真实用户，就应该把基础设施当成成本，而不是负担。

早期可以从每月几十元的轻量服务器开始。用户增长后，再逐步拆分数据库、对象存储、队列和监控。
