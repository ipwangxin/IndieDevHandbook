# 怎么获得第一批用户

> *字数：约 1780 字；预计阅读时间：4 分钟。*

我之前在知乎回答独立开发和推广相关的问题，遇到过一个很典型的情况：有人点赞，有人收藏，也有人关注，但没有多少人真的去提交 Issue。

一开始我会觉得是不是项目吸引力不够。后来想明白了：收藏和点赞说明“这个话题有兴趣”，但从知乎跳到 GitHub，再按模板提 Issue，这个动作太重了。

这就是冷启动里很常见的误判：

> 你以为用户没有需求，其实可能只是你给他的下一步太难。

## 第一批用户不是自然来的

产品上线后，不会因为它存在就有人来。

尤其是早期，你不能只发一句“我做了一个产品，欢迎体验”。这句话对你很重要，对用户不一定重要。用户更关心的是：

- 这和我有什么关系？
- 它能帮我省什么？
- 我现在要做什么？
- 做完之后能得到什么？

所以推广不是把链接甩出去，而是设计一条从“看到”到“行动”的路径。

## 先把动作变轻

如果用户只是刚刚感兴趣，不要一上来就让他完成很重的动作。

比如早期收集失败项目时，我不应该只说：

> 欢迎去 GitHub 提 Issue。

更轻的说法应该是：

> 如果你有做了一半但没继续的项目，评论一句项目名和为什么停了，我可以帮你整理成案例。

动作从 5 分钟变成 10 秒，转化就可能完全不同。

早期可以用这些轻动作：

- 评论一个关键词。
- 留邮箱。
- 加群。
- 填一个极简表单。
- 上传一个样本。
- 免费试用一次。
- 回复“我想看这个”。

## 免费渠道怎么用

### 知乎

知乎适合写经验、复盘和问题回答。

不要只写“我做了一个产品”。更好的角度是：

- 我遇到了什么真实问题。
- 我怎么验证这个需求。
- 为什么点赞收藏没有转化。
- 我从失败项目里看到什么共性。
- 独立开发第一批用户到底怎么找。

知乎的好处是长尾搜索，坏处是转化路径容易断。所以结尾动作要轻，不要塞太多链接。

### V2EX

V2EX 适合开发者产品、开源项目、工具站。

这里用户很讨厌硬广，但愿意看真实过程。你可以讲技术栈、动机、当前问题、希望大家反馈什么。

不要包装得太满。越像真实开发者发的帖子，越容易得到具体反馈。

### 即刻

即刻适合 build in public。

但 build in public 不是每天发“今天又写了两小时代码”。更有价值的是公开你的判断：

- 为什么砍掉这个功能？
- 哪个渠道试了没用？
- 哪个标题带来了更多评论？
- 用户反馈让你改了什么？

### 小红书

小红书更吃场景和结果。

如果你做的是效率工具、AI 工具、副业工具、模板类产品，不要讲技术，讲“我用它解决了什么问题”。

截图、对比、前后变化，比长篇解释更重要。

### GitHub

GitHub 适合开发者受众和开源项目。

README 要像落地页一样写清楚：

- 这是什么？
- 适合谁？
- 怎么体验？
- 当前做到哪一步？
- 怎么参与？

不要假设别人会从代码里理解你的项目。

## 要不要买广告

早期不要指望广告救产品。

可以买，但要把它当成买数据，而不是买增长。

可以买广告的前提：

- 页面能讲清楚价值。
- 有明确 CTA。
- 有访问统计。
- 能区分来源和转化。
- 自然流量已经证明有人感兴趣。
- 你知道一次转化大概值多少钱。

预算可以很小：

```text
每天 20-100 元，连续 3-7 天
```

如果广告带来点击，但没人行动，不要急着加预算。先改标题、落地页、CTA 和目标人群。

## 自媒体账号要不要先做

如果你的产品需要信任、教育市场、长期内容，建议做。

比如独立开发教程、AI 工具、SEO 工具、模板、B2B 方法论产品，都适合用内容先聚人。

但不要把自媒体做成另一个大项目。最低程度就够：

- 一个主平台。
- 每周 2-3 条短内容。
- 每周 1 篇长内容。
- 每条内容围绕用户问题。
- 结尾只给一个轻动作。

你不是在做媒体公司，你是在给产品找种子用户。

## 怎么找重叠人群账号

不要只看粉丝量。

你要看评论区有没有你的目标用户。

步骤很简单：

1. 列目标用户关键词。
2. 在知乎、小红书、B站、即刻、公众号搜索。
3. 找持续发相关内容的账号。
4. 看评论区有没有具体问题。
5. 看他们是否推荐过类似工具、课程、模板或服务。
6. 先互动、评论、引用，再考虑合作。

一个 5000 粉但评论全是目标用户的账号，可能比一个 10 万粉但泛泛娱乐的账号更值钱。

## 我现在会按这个顺序冷启动

```text
写一篇问题型内容
-> 看评论和私信
-> 把感兴趣的人导到群或表单
-> 找 10 个愿意细聊的人
-> 做一个非常小的 MVP
-> 用真实反馈改页面和功能
-> 再考虑广告、合作或联盟
```

第一批用户不需要很多。

最好的第一批用户，是愿意说话的人。他们会告诉你哪里看不懂、哪里不信、哪里真的有用、哪里只是你自己觉得重要。

冷启动不是让所有人都来，而是先找到一小撮真的在乎的人。
