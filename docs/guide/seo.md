# SEO 基础

> *字数：约 2050 字；预计阅读时间：5 分钟。*


SEO 对独立开发者很重要，因为它能把一次写作变成长期入口。

知乎、V2EX、即刻、小红书的流量通常来得快、走得也快；搜索流量慢，但更稳定。

## 什么时候开始做 SEO

越早越好，但早期只做基础。

不要一开始买昂贵 SEO 工具，也不要堆关键词。先确保搜索引擎能正常理解你的网站。

## 必做基础

- 每个页面有清楚的 title。
- 每个页面有 description。
- URL 简短可读。
- 首页第一屏说明你解决什么问题。
- 有 sitemap.xml。
- 有 robots.txt。
- 移动端可读。
- 页面加载速度不要太慢。
- 页面之间有清楚内链。
- 提交到搜索资源平台。

## title 和 description 怎么写

不要把每一页都写成品牌名。

更好的写法是：

```text
用户问题 + 具体场景 + 品牌/站点名
```

比如：

```text
独立开发怎么获得第一批用户 - 国内独立开发冷启动手册
VitePress 怎么部署上线 - 国内独立开发冷启动手册
个人开发者怎么接入支付 - 国内独立开发冷启动手册
```

description 不要堆关键词，直接说明这一页能解决什么问题：

```text
整理独立开发者从上线、统计、评论、推广到第一批反馈的行动路径，适合刚发布 MVP 的小产品。
```

用户在搜索结果里看到这句话，要能判断：点进来是不是能解决我的问题。

## 关键词怎么找

从用户问题开始，不要从产品名开始。

比如你做独立开发手册，用户更可能搜索：

- 独立开发怎么开始
- 独立开发怎么推广
- 独立开发怎么找需求
- 独立开发怎么接支付
- Vercel 怎么部署
- 个人开发者怎么收款

每个关键词背后都应该是一篇真正能解决问题的文章。

早期可以从这些地方找关键词：

- 知乎问题标题。
- 百度和 Google 搜索下拉。
- 竞品文档目录。
- V2EX、即刻、小红书评论区。
- 你自己收到的评论和私信。
- Google Search Console 里已经有曝光但点击低的词。

不要一开始追“大词”。比如“独立开发”太大，“独立开发怎么推广”“独立开发怎么接支付”“小产品上线后第一周做什么”更容易写出有用内容。

## 免费书为什么适合 SEO

免费书天然适合 SEO，因为它有结构化目录：

```text
首页
-> 章节
-> 子主题
-> 工具清单
-> 案例
```

每一章都可以是一个搜索入口。

你的目标不是让所有人从首页进入，而是让用户搜索某个具体问题时，直接进入对应章节。

## 内链怎么做

内链不是为了凑链接，而是帮用户下一步继续走。

比如：

- 用户读“怎么部署上线”，下一步可以去“上线后一周行动清单”。
- 用户读“上线后一周行动清单”，下一步可以去“数据统计和分析”。
- 用户读“怎么获得第一批用户”，下一步可以去“流量联盟”。
- 用户读“什么时候应该放弃”，下一步可以去“失败项目复盘”。

每篇文章至少放 2-3 个自然内链。不要在一段话里塞十几个链接，那会像站群。

## SEO 内容写法

一篇 SEO 文章可以按这个结构写：

```md
# 用户正在搜索的问题

先直接回答。

## 什么情况下适用

说明边界。

## 具体怎么做

给步骤和例子。

## 常见坑

提醒风险。

## 下一步

引导到相关章节、工具、案例或社群。
```

不要写很多空泛概念。搜索用户通常带着问题来，越快给出可执行答案越好。

## 上线后看什么

接入 Google Search Console 后，不要每天只看总点击。

更应该看：

- 哪些页面开始有曝光。
- 哪些关键词有曝光但没人点。
- 哪些页面排名在 8-20 名，适合补内容。
- 哪些关键词和你原本想的不一样。
- 哪些章节被搜索用户直接进入。

如果一个页面有曝光但点击低，先改标题和描述。

如果一个页面有点击但停留短，说明内容没有接住搜索意图，要补步骤、例子、截图或常见坑。

## 免费书怎么拆成长尾文章

每一章都可以继续拆。

比如“怎么部署上线”可以拆成：

- VitePress 怎么部署到 Vercel
- VitePress 怎么绑定自定义域名
- 独立开发生产服务器怎么选
- 香港服务器适合什么场景
- 小产品上线后要不要买流量

拆文章时不要复制粘贴一遍正文。长尾文章应该更具体、更可操作，最好能回答一个单独问题。

## 怎么借 AI 和 vibe coding 的搜索热度

AI 和 vibe coding 是热点，但不要只追热点词。

更好的写法是把热点和真实问题结合：

- vibe coding 适合做什么 MVP？
- AI 写代码做出来的产品为什么没人用？
- 独立开发者怎么用 AI 做第一版？
- 用 AI 做 SaaS 时哪些地方不能偷懒？
- AI 产品怎么定价才能覆盖成本？
- AI 生成代码上线前要检查什么？

这些标题既能接住 AI 热点，也不会偏离这本书的主线。

如果一篇文章只是“我用 AI 一天做了一个产品”，热度可能来得快，但不一定能沉淀长期搜索。更适合长期 SEO 的内容，是把过程拆成可复用的方法、检查清单和失败经验。

## SEO 不是唯一渠道

SEO 起效慢，不能替代冷启动。

更现实的组合是：

```text
知乎 / V2EX / 即刻 / 小红书 获取短期反馈
免费书 / 工具页 / 案例库 承接长期搜索
社群 / 邮件列表 沉淀高意向用户
```

短期靠主动推广，长期靠内容资产。
