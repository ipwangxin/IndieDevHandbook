# SEO 基础

> *字数：约 519 字；预计阅读时间：2 分钟。*


SEO 对独立开发者很重要，因为它能把一次写作变成长期入口。

知乎、V2EX、即刻、小红书的流量通常来得快、走得也快；搜索流量慢，但更稳定。

## 什么时候开始做 SEO

越早越好，但早期只做基础。

不要一开始买昂贵 SEO 工具，也不要堆关键词。先确保搜索引擎能正常理解你的网站。

## 必做基础

- 每个页面有清楚的 title。
- 每个页面有 description。
- URL 简短可读。
- 首页第一屏说明你解决什么问题。
- 有 sitemap.xml。
- 有 robots.txt。
- 移动端可读。
- 页面加载速度不要太慢。
- 页面之间有清楚内链。
- 提交到搜索资源平台。

## 关键词怎么找

从用户问题开始，不要从产品名开始。

比如你做独立开发手册，用户更可能搜索：

- 独立开发怎么开始
- 独立开发怎么推广
- 独立开发怎么找需求
- 独立开发怎么接支付
- Vercel 怎么部署
- 个人开发者怎么收款

每个关键词背后都应该是一篇真正能解决问题的文章。

## 免费书为什么适合 SEO

免费书天然适合 SEO，因为它有结构化目录：

```text
首页
-> 章节
-> 子主题
-> 工具清单
-> 案例
```

每一章都可以是一个搜索入口。

你的目标不是让所有人从首页进入，而是让用户搜索某个具体问题时，直接进入对应章节。

## SEO 内容写法

一篇 SEO 文章可以按这个结构写：

```md
# 用户正在搜索的问题

先直接回答。

## 什么情况下适用

说明边界。

## 具体怎么做

给步骤和例子。

## 常见坑

提醒风险。

## 下一步

引导到相关章节、工具、案例或社群。
```

不要写很多空泛概念。搜索用户通常带着问题来，越快给出可执行答案越好。

## SEO 不是唯一渠道

SEO 起效慢，不能替代冷启动。

更现实的组合是：

```text
知乎 / V2EX / 即刻 / 小红书 获取短期反馈
免费书 / 工具页 / 案例库 承接长期搜索
社群 / 邮件列表 沉淀高意向用户
```

短期靠主动推广，长期靠内容资产。
