# 上架门槛总览

> *字数：约 2070 字；预计阅读时间：5 分钟。*


这一章是合规地图，不是法律、税务或经营建议。备案、许可、隐私、支付、广告、应用商店审核和平台规则都会变化，真正上线前应以官方文档、平台后台提示、接入服务商要求和专业顾问意见为准。

如果你的项目涉及医疗、金融、教育、招聘、未成年人、投资、算命占卜、内容社区、用户上传、跨境支付或高风险 AI 生成内容，不要只靠经验帖判断，至少要单独查一次对应平台和监管要求。

独立开发常见发布形态有四类：

- 网站
- 微信小程序
- 国内 App
- 海外 App / 海外网站

它们的门槛不一样。网站最灵活，小程序自带微信生态但审核更明确，App 分发能力强但上架和合规更重。海外整体流程看起来更轻，但同样有平台政策、隐私、支付、税务和内容审核。

## 先看总览

| 形态 | 常见门槛 | 适合什么阶段 |
| --- | --- | --- |
| 网站 | 域名、服务器、备案/不备案、隐私政策、支付合规 | MVP、SEO、工具站、SaaS |
| 微信小程序 | 小程序账号、主体、类目、备案、微信审核、支付/广告能力 | 国内用户、微信场景、轻应用 |
| 国内 App | 应用市场账号、软著可能要求、隐私合规、App 备案、平台审核 | 重度移动端、需要原生能力 |
| iOS App Store | Apple Developer Program、App Review、隐私信息、IAP 规则 | 海外/国内 iOS 用户 |
| Google Play | Play Console、开发者验证、政策审核、Billing 规则 | 海外 Android 用户 |
| 海外网站 | 域名、服务器、隐私政策、支付和税务 | 出海 SaaS、内容站、工具站 |

## 上线路径选择表

| 产品形态 | 常见门槛 | 常见低门槛入口 |
| --- | --- | --- |
| 纯内容站 | 域名、部署、备案/不备案、版权、隐私 | 静态站、海外部署、免费书 |
| 工具站 | 文件/数据安全、统计、隐私、备案 | 网页 Demo、表单、免费工具 |
| AI 工具 | 模型 API、用户上传、生成内容、成本控制 | 限额试用、人工审核、免责声明 |
| 微信小程序 | 主体、类目、备案、审核、支付/广告能力 | 个人主体轻工具、内容入口 |
| iOS App | Apple 开发者账号、App Review、IAP、隐私 | TestFlight、先做网站验证 |
| Android App | Play Console/国内市场、备案、Billing、隐私 | APK 内测、网站/PWA 先验证 |
| B2B SaaS | 合同、发票、数据安全、企业支付、权限 | 落地页 + 人工服务 + 试用 |
| 社区/UGC | 内容审核、举报、用户协议、风控 | 先做内容精选或表单收集 |
| 电商/交易 | 主体、支付、售后、商品资质、平台责任 | 先做需求收集或人工撮合 |

## 常见误区

### 有软著就能上架

不一定。软著是权属证明，不等于备案、许可证或平台审核通过。

### ICP 备案等于 ICP 经营许可证

不等于。ICP备案是基础登记，ICP 经营许可证是经营性互联网信息服务可能涉及的许可。

### 海外部署就完全不用合规

不对。海外部署可能不需要国内 ICP 备案，但仍然有隐私、支付、税务、平台政策和目标市场法规。

### App 免费就没有平台费用

不完全。免费 App 通常不按下载向平台付费，但开发者账号、审核、广告 SDK、服务器、统计、客服和隐私合规仍然有成本。

### 个人小程序可以随便收款

通常不行。个人主体小程序商业能力有限，微信支付、类目和交易能力要看主体和平台规则。

### AI 工具不用管用户上传数据

不对。用户上传的合同、图片、简历、报价、音频、病历等都可能包含敏感信息。你需要说明是否存储、是否用于训练、是否传给第三方模型、如何删除。

### 接了广告就一定赚钱

不一定。广告收入取决于有效曝光、eCPM、填充率、用户地区、广告类型和留存。低流量工具接广告可能收入很少，还会伤害体验。

## 最后怎么选

这里只列门槛，不做主观推荐。

如果只是验证需求，通常可以先用低门槛入口；如果已经有真实用户和收入，再补齐正式分发、备案、支付和合规。
